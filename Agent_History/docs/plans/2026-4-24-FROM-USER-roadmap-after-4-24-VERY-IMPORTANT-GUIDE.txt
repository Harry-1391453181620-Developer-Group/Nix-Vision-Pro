These 3 phases aims to rigorously develop and validate a novel attractor-based loss (Ω-loss) within a controlled engineering pipeline, starting from an existing CNN classifier and progressively evolving toward a small-scale vision-language model (VLM). The core hypothesis is that enforcing a self-consistency constraint of the form h ≈ T(h) improves representation stability, generalization, and ultimately enables a transition from purely discriminative models to semantically grounded dynamical systems.

Phase1: Phase 1 focuses exclusively on validating the Ω-loss within the existing CNN framework without introducing architectural changes. The model remains a standard convolutional backbone followed by global pooling and a classification head producing logits. A lightweight transformation function T(h), implemented as a shallow MLP with one or two layers and normalization, is introduced to operate on the pooled feature vector h. The total loss is defined as L_total = L_CE_mix + λ · L_attr, where L_CE_mix is the existing training loss incorporating cross-entropy, label smoothing, and stochastic mixup/cutmix augmentation, and L_attr = ||h − sg(T(h))||² with sg denoting stop-gradient to prevent collapse. All existing training configurations, including optimizer (AdamW), cosine learning rate schedule, augmentation pipeline, EMA, AMP, and batch size, must remain unchanged to ensure isolation of the loss effect. Experiments must include at least three configurations: λ = 0 (baseline), λ = 0.05, and λ = 0.1. For each run, the following metrics must be recorded: training loss, validation loss, training accuracy, validation accuracy, generalization gap, and qualitative behavior of convergence (smoothness, oscillation, or divergence). Additional diagnostics must include monitoring the variance of h across the batch to detect representation collapse. Phase 1 is considered successful if any of the following conditions are met: validation accuracy improves over baseline, convergence becomes more stable, or the generalization gap decreases. If none of these conditions are met, λ and the complexity of T(h) must be adjusted while preserving all other variables.
Phase2: Phase 2 introduces structural changes to lift the representation from a single global vector to a structured token space, enabling the Ω-loss to operate on higher-dimensional dynamics. The CNN backbone is modified to output a spatial feature map which is reshaped into a sequence of tokens H ∈ ℝ^{N×D}. A lightweight transformer block (single layer, minimal heads, low dimensionality) is applied to produce T(H). The classification head operates on either a pooled token or a designated CLS token. The Ω-loss is extended to L_attr = ||H − sg(T(H))||², applied across all tokens. The total loss remains L_total = L_CE_mix + λ · L_attr, with λ initialized from the best value found in Phase 1. Experiments must include three configurations: tokenized model without Ω-loss, tokenized model with Ω-loss, and the best Phase 1 CNN configuration for reference. Metrics remain identical to Phase 1, with additional analysis of token-level statistics such as inter-token variance and stability across training iterations. Phase 2 is considered successful if the tokenized model with Ω-loss outperforms the tokenized baseline in validation accuracy or demonstrates improved stability, even if accuracy gains are marginal. Failure conditions include training instability, collapse of token diversity, or degradation relative to both baselines, in which case transformer complexity or λ must be reduced.
Phase3: Phase 3 introduces a proto-VLM structure by aligning visual representations with a semantic embedding space, enabling the attractor mechanism to operate at the semantic level. A learnable class embedding matrix E ∈ ℝ^{C×D} is introduced, representing pseudo-text embeddings. The model computes logits via similarity between image-derived representations and E, typically through a dot product between a pooled token (or CLS token) and class embeddings. In addition to the existing Ω-loss on tokens, a semantic attractor mechanism is defined by computing H' = Attention(H, E), where Attention denotes a standard scaled dot-product attention between tokens and class embeddings. A new loss term is introduced: L_semantic_attr = ||H − sg(H')||². The total loss becomes L_total = L_CE + λ₁ · L_attr + λ₂ · L_semantic_attr, where λ₁ is inherited from Phase 2 and λ₂ is introduced with small initial values (e.g., 0.01–0.05). Training must again preserve all other configurations unless instability is observed. Experiments must include ablations: without Ω-loss, with only token-level Ω-loss, and with both token and semantic attractor losses. Metrics include all previous measurements, with additional evaluation of embedding quality, such as clustering of class representations in embedding space and robustness to input perturbations. Phase 3 is successful if the addition of semantic attractor loss improves validation accuracy, enhances robustness, or produces more structured embedding distributions.

Across all phases, strict experimental discipline must be maintained. Only one major variable is introduced per phase, and all comparisons must be controlled. Early stopping, checkpointing, and logging must remain consistent. Any observed improvement must be reproducible across multiple runs with fixed seeds. The project must not introduce complex memory systems, dynamic attractor banks, or advanced mechanisms such as engram-style memory until all three phases demonstrate stable and positive results.

The final outcome of the phases is a validated pipeline demonstrating that Ω-loss can first stabilize scalar representations, then structured token representations, and finally semantic representations within a proto-VLM framework. This establishes a foundation for future extensions into full attractor-based memory systems and dynamical architectures.
